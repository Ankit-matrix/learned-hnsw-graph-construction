# Learned HNSW Graph Construction

> **Learning Graph Construction Parameters for Hierarchical Navigable Small Worlds (HNSW) in Approximate Nearest Neighbor Search**

A research implementation exploring **machine learning-guided graph construction** for **Hierarchical Navigable Small World (HNSW)** indices. This project replaces fixed graph construction heuristics with learned decisions and Bayesian optimization to improve recall while maintaining scalable graph construction.

---

## Overview

Approximate Nearest Neighbor (ANN) search is a fundamental component of modern machine learning systems powering

* Semantic Search
* Retrieval-Augmented Generation (RAG)
* Recommendation Systems
* Vector Databases
* Image Retrieval

Among ANN algorithms, **Hierarchical Navigable Small Worlds (HNSW)** consistently provides one of the best trade-offs between search accuracy and latency.

Traditional HNSW construction relies on manually selected heuristics including

* Number of neighbors (**M**)
* Construction beam width (**ef_construction**)
* Random layer assignment
* Distance-based neighbor selection

These heuristics are dataset-agnostic and do not exploit query workload information.

This project investigates whether graph construction itself can be **learned from query examples**, allowing HNSW to adapt to the underlying data distribution.

---

# Motivation

Standard HNSW assumes that every dataset should use identical construction heuristics.

However,

* different datasets exhibit different manifold structures,
* some nodes are significantly more important for navigation,
* optimal graph connectivity depends on the query distribution.

Instead of manually tuning these parameters, we learn them directly from data.

---

# Contributions

This project introduces a two-level learning framework.

## 1. Bayesian Hyperparameter Optimization

Automatically learns

* M
* ef_construction

using Bayesian Optimization (Tree-structured Parzen Estimator via Optuna).

Best configuration discovered

```
M = 48
ef_construction = 400
```

---

## 2. Learned Neighbor Selection

Instead of selecting neighbors purely by Euclidean distance, an XGBoost classifier learns which candidate edges improve future ANN search performance.

Node features include

* Euclidean distance
* Local density
* Traversal frequency
* Directional diversity

---

## 3. Query Trace Learning

The framework records graph traversal paths generated by thousands of ANN queries and learns which nodes are frequently involved in successful search paths.

---

## 4. Large-Scale Engineering Optimizations

The initial learned implementation was prohibitively slow.

This project introduces four exact optimizations that reduce construction time dramatically while preserving correctness.

# Methodology

## Level 1 — Bayesian Optimization

Search space

```
M ∈ {8,12,16,24,32,48}

ef_construction ∈ {100,150,200,300,400}
```

Objective

Maximize

```
Recall-QPS AUC
```

Optimization library

```
Optuna
```

Algorithm

```
Tree-structured Parzen Estimator (TPE)
```

---

## Level 2 — Learned Neighbor Selection

Each candidate edge is represented using

* Euclidean Distance
* Local Density
* Traversal Frequency
* Directional Diversity

An XGBoost classifier predicts whether the edge should exist in the final graph.

Unlike vanilla HNSW, edge selection is therefore driven by learned routing quality rather than only geometric proximity.

---

# Feature Engineering

Each node stores

* Traversal Frequency
* Local Density
* Vector Norm
* First Five PCA Components

Each candidate edge stores

* Distance
* Neighbor Density
* Traversal Score
* Directional Diversity

# Performance Optimizations

The naive implementation required

```
56 minutes
```

for

```
2,000 nodes
```

which projected to

```
27+ hours
```

for the complete dataset.

Four optimizations solved this bottleneck.

---

## Optimization 1

### Distance Cache Expansion

```
500K
        ↓
3 Million Entries
```

Cache hit rate

```
17.6%
        ↓
78.9%
```

---

## Optimization 2

### Early Beam Search Termination

Beam search exits after

```
30
```

non-improving expansions.

---

## Optimization 3

### Lazy XGBoost Evaluation

Learned scoring is skipped for layer 0.

Only upper layers invoke the model.

---

## Optimization 4

### Candidate Prefiltering

Instead of evaluating

```
400 candidates
```

the classifier evaluates only

```
Top-20 nearest candidates
```
